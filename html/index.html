Wellstarter

Software engineer, 2018-

Trondheim, Norway


Wellstarter is an oil and gas service company developing a system for improving the observability of inflow rates within oil wells. This is done by generating signals in the form of heat pulses in oil wells which are propagated and affected by the inflow rates upstream and downstream of the source location which can be measured at the "heel" of the well.


I started working part-time at Wellstarter to develop an ERP solution. Being an oil and gas service company, Wellstarter is expected to provide detailed information on every part of its product such as material certificates, logistics tracking, production process and product testing.


During my final year at NTNU i cooperated with Wellstarter on both my project and master thesis. Sjur Mo, Senior Staff Production Engineer and Ph.D in applied physics, supervised and offered great insight into numerically solving physical systems.

mLink Monitor

Starting in Moen Marin i quickly became responsible for our edge offering, mLink Monitor. This solution was plagued by badly integrated and unsuitable third party software meant to aid in transfer of data and storage in the cloud. With support from a senior developer I provided the outline for a dockerized architecture with the aim of having a set of microservices, each performing a single task. The techonologies included are mainly:
  • Revolution Pi Core & Compact
  • Docker
  • PostgreSQL & TimescaleDB
  • Golang
  • Canbus
  • Modbus
  • MQTT
Golang was chosen as the language to use for each service due to its very slim hardware footprint and excellent multithreading capabilities. With the help of a team of students working part-time we built modules to read data from each available data source onboard the ships such as engines, generators, weather station, navigation systems and more.

Athos

Wellstarter needed a system to individually track each component in systems sold to customers and installed in oil wells. I've made an attempt at creating such a system using Python Django along with PostgreSQL. The full list of technologies in the stack are:
  • Google cloud
  • Docker
  • Gunicorn
  • Python Django
  • PostgreSQL
  • MS Graph
  • Java script + JQuery
  • CSS + bootstrap 4
  • HTML5
Central to the backend is a recursive table structure defining individual components and assemblies. A corresponding recursive structure represents actual components that can be acquired throigh purchase orders or work orders. Components are tracked by connecting with a timestamp and an event which can for example be the component being destroyed in a test, made part of an assembly or passing through a country. Purchase orders for assemblies can include components sent from Wellstarter storage. A system for registering tests and tests results is also implemented including a micro-service for uploading live data from experiments and tests. Other features

I started building a system at a family cabin in 2017 with a hardware stack constisting of a raspberry pi, a usb z-wave stick and a various z-wave compatible sensors. I set up homeassistant using Docker and published the site using Nginx. In the years since the system has grown to encompass 2 cabins and my family home with the list of hardware now being:
  • 2 Asus barebone machines
  • 2 UPS'es
  • 7 raspberry pi's
  • 5 pi cameras
  • 7 multisensors (temperature, PIR, light, pressure, humidity)
  • 8 smoke detectors, 5 with temperature sensor
  • 6 wall plugs
  • 12 thermostats
  • 1 weather station
The 5 pi's running cameras either livestream video or capture image/video on request. One of the cabins depend on gsm/4g making livestreaming a bit costly datawise. The 3 homes each have a dedicated server with the 2 cabins having UPS'es and using barebone machines rather than pi's. This is partly due to the weakness of the sd-card, causing the pi's to suddenly crash after a few years, always when noone are present. The software stack for the servers at each home are:
  • Docker
  • Homeassistant
  • PostgreSQL
  • Nginx (except for the cabin on mobile net)
